STEP-BY-STEP: BUILD IN 30 MINUTES (EXPRESS PATH)
================================================================================

STEP 1: PREPARE (5 minutes)
-------
1. Print or bookmark: MODEL_BUILD_CHECKLIST.txt
2. Open VS Code with all reference docs side-by-side
3. Have Power BI Desktop and SQL Server ready
4. Keep DAX_MEASURES.txt open for copy-paste

STEP 2: IMPORT DATA (5 minutes)
-------
1. Power BI Desktop -> Get Data -> SQL Server
2. Server: localhost
3. Database: AdventureWorksDW2022
4. Select all 17 tables (13 dimensions + 4 facts)
5. Click Load

STEP 3: MARK DATE TABLE (1 minute)
-------
1. Data tab -> Right-click DimDate
2. Mark as date table -> Select FullDateAlternateKey
3. Done!

STEP 4: CREATE RELATIONSHIPS (10 minutes)
-------
1. Model tab
2. Use the checklist to drag relationships:
   - 7 from FactInternetSales
   - 8 from FactResellerSales
   - 7 dimension-to-dimension
3. Set ACTIVE/INACTIVE as noted
4. Verify no red X errors

STEP 5: CREATE MEASURES TABLE & MEASURES (5 minutes)
-------
1. Data tab -> New Table -> Name: _Measures
2. Formula: _Measures = ROW("Placeholder", 1)
3. For each measure in DAX_MEASURES.txt:
   - New Measure
   - Copy-paste expression
   - Press Enter
4. Repeat for all 22 measures

STEP 6: HIDE KEYS & FORMAT (2 minutes)
-------
1. Data tab -> Hide all *Key columns
2. Format *Amount columns as Currency
3. Format *Pct columns as Percentage

STEP 7: TEST (1 minute)
-------
1. Report tab -> Create Card visual
2. Add [Total Sales Amount] measure
3. Should show ~$61.5M
4. Create slicer to test filters

STEP 8: SAVE (1 minute)
-------
1. File -> Save As
2. Name: AdventureWorks_Semantic_Model
3. Type: Power BI (*.pbix)
4. Done!

TOTAL TIME: ~30 minutes (EXPRESS BUILD)

1. SQL SERVER MUST BE ACCESSIBLE

   - Check: SQL Server name is correct (localhost or [MACHINE]\[INSTANCE])
   - Check: AdventureWorksDW2022 database exists

2. FOLLOW RELATIONSHIP CARDINALITY

   - Single direction only
